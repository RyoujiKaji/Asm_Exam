# **Ассемблер** (от англ. **assembler** — сборщик) 
— транслятор программы из текста на языке ассемблера в [[4. Система команд. Основные этапы выполнения команды#**Система команд процессора** (Instruction Set Architecture, ISA)|машинный код]]

# **Язык ассемблера**
— язык программирования низкого уровня, который представляет собой текстовую форму машинного кода. Если машинный код состоит из нулей и единиц (или шестнадцатеричных чисел), которые понимает процессор, то ассемблер заменяет эти числа на понятные человеку сокращения — *мнемоники*.
##### *Мнемоники* 
— короткие символические имена, которые представляют машинные команды

Некоторые сферы применения ассемблера:

- **Разработка операционных систем** (ядро, загрузчики). Ассемблер используется для написания аппаратно-зависимых участков кода, например, загрузчика ОС, HAL и ядра. 
- **Программирование микроконтроллеров** с ограниченными ресурсами, где каждый байт памяти и каждый такт процессора имеют значение. Ассемблер удобен для программирования микроконтроллеров, так как одна его команда транслируется в одну команду в двоичном коде. 
- **Высокопроизводительные вычисления** и оптимизация критических участков кода. Ассемблер используется для создания оптимизированных библиотек и функций, в критических местах — для ручной оптимизации с использованием специфических инструкций процессора (например, SSE/AVX для x86 или NEON для ARM). 
- **Кибербезопасность** — знание ассемблера важно для специалистов по анализу вредоносного ПО и реверс-инжинирингу. Когда исходный код недоступен, дизассемблированный код становится единственным источником информации о работе программы.
- **Мультимедиа** — оптимизация обработки аудио/видео, например, в кодеках, графических библиотеках, игровых движках

# **Структура программы на ассемблере** 

На примере masm
#### **MASM (Microsoft Macro Assembler)** 
— ассемблер для процессоров [[4. Система команд. Основные этапы выполнения команды#**Система команд процессора** (Instruction Set Architecture, ISA)|семейства x86]].

Структура программы на языке ассемблера для архитектуры Intel (x86) определяется [[6.1. Директивы ассемблера. Операторы ассемблера.#**Директива ассемблера** (псевдооператор)|директивами]], которые указывают компилятору (ассемблеру), как организовать код и данные в памяти.

Типовая структура программы состоит из трех основных частей: **[[5. Организация памяти. Модели памяти. Способы адресации#**Модель памяти**|модели памяти]]**, **[[5. Организация памяти. Модели памяти. Способы адресации#**Основные сегменты**|сегментов]]** и **точки входа**.

```masm
.586 ;какой набор инструкций будет использоваться
.MODEL     flat, stdcall
OPTION     CASEMAP:NONE    ; описание  модели памяти      ;чувствительность к регистру 

.CONST  ; сегмент констант 
    RequestA DB 13,10,'Input A',13,10,0 ; строка

.DATA ; сегмент инициализированных данных
    ResStr DB 16 DUP (' '), 0

.DATA? ; сегмент неинициализированных данных
    A SWORD ?
    BufferA DB 10 DUP (?)

.CODE ;сегмент кода
Start:     
...
End Start ;конец программы + метка точки входа
```
