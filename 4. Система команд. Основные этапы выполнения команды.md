# **Система команд процессора** (Instruction Set Architecture, ISA)
— интерфейс между аппаратным и программным обеспечением, определяющий набор инструкций, которые может выполнять процессор. 

В систему команд входят типы данных, регистры, модель памяти, режимы адресации и формат инструкций. Система команд жёстко связана с конкретным типом процессора.

Система команд определяет:
- **Словарь операций:** что процессор умеет делать (сложение, логическое И, переход).
- **Таблицу кодирования:** какое именно двоичное число (опкод) соответствует каждой операции.
- **Синтаксис ассемблера:** какие текстовые сокращения (мнемоники) должны использоваться для этих операций в [[6. Назначение ассемблеров. Структура программы.|языке ассемблера]].

# **Структура команды**

Любая команда в системе команд процессора состоит из двух частей:
1. **Операционная** — показывает одно действие из списка допустимых, которое нужно выполнить с данными. Каждая операция имеет свой уникальный код.
2. **Адресная** — показывает, где хранятся эти данные и куда записать результат операции. Основу адресной части составляет операнд — сущность, над которой производится операция.

Типовая команда, в общем случае, должна указывать:
- подлежащую выполнению операцию;
- адреса исходных данных (операндов), над которыми выполняется операция;
- адрес, по которому должен быть помещен результат операции.

# **Основные этапы выполнения команды**

Центральный процессор выполняет каждую команду за несколько шагов. Он делает следующее:

1. ***Вызывает следующую команду из памяти и переносит ее в регистр команд.***
2. ***Меняет положение счетчика команд***, который после этого указывает на следующую команду.
3. ***Определяет тип вызванной команды.***
4. ***Если команда использует слово из памяти, определяет, где находится это слово.***
5. ***Переносит слово***, если это необходимо, ***в регистр ЦП***.
6. ***Выполняет команду***.
7. ***Переходит к шагу 1***, чтобы начать выполнение следующей команды.
